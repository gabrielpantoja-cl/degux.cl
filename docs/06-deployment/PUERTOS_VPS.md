**Causa:**


El healthcheck espera `/api/health` pero la ruta devuelve 404.

**Impacto:**
- Docker marca el contenedor como unhealthy
- nginx-proxy sigue proxy-pasando tr√°fico (funciona pero con warning)

**Soluci√≥n pendiente:**
